## 为什么需要费用预警

### 风险场景

1. **被恶意刷流量** - 攻击者大量请求您的资源
2. **配置错误** - 某个设置导致异常流量
3. **预估不足** - 实际用量超出预期

### 费用构成

| 服务 | 主要计费项 | 单价（按量付费） |
|------|-----------|----------------|
| OSS | 外网流出流量 | 0.5元/GB |
| OSS | 请求次数 | 0.01元/万次 |
| CDN | 流量费 | 0.04元/GB |
| CDN | HTTPS请求 | 0.05元/万次 |

**举例**：被攻击产生100GB流量 = 50元

## 设置预算预警

### 步骤1：访问费用中心

1. 登录阿里云控制台
2. 访问：https://费用中心.aliyun.com
3. 或点击右上角「费用」→ 「费用中心」

### 步骤2：进入预算管理

1. 左侧菜单点击「预算管理」→「用量预算」
2. 点击「创建预算」

### 步骤3：创建预算规则

配置信息：

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| **预算名称** | 月度网站费用预算 | 自定义名称 |
| **产品** | 对象存储OSS + CDN | 选择相关产品 |
| **地域** | 全部地域 | 或选择特定地域 |
| **预算类型** | 金额 | 按费用金额 |
| **预算周期** | 每月 | 或选择其他周期 |
| **预警阈值** | 50% → 80% → 100% | 多档预警 |
| **预警方式** | 邮件 + 短信 | 建议开启短信 |

### 步骤4：设置阈值

推荐配置三档预警：

```
第一档：50% 提醒
  - 例如：月预算100元，费用达50元时提醒
  - 提醒内容："您的网站费用已达预算的50%，请关注"

第二档：80% 警告
  - 例如：月预算100元，费用达80元时警告
  - 提醒内容："您的网站费用已达预算的80%，请尽快检查"

第三档：100% 紧急
  - 例如：月预算100元，费用达100元时紧急
  - 提醒内容："您的网站费用已达预算上限，建议立即检查"
```

## 设置OSS流量预警

### 方法1：使用OSS监控报警

#### 步骤1：进入OSS监控

1. OSS控制台：https://oss.console.aliyun.com/
2. 进入您的存储桶
3. 左侧菜单「数据监控」→「基础监控」

#### 步骤2：设置报警规则

1. 点击「报警规则」→「创建报警规则」
2. 配置：

| 配置项 | 推荐值 |
|--------|--------|
| **规则名称** | OSS流量预警 |
| **监控指标** | 公网流出流量 |
| **统计周期** | 1小时 |
| **条件** | > 设定值 |
| **阈值** | 1GB/小时（或根据实际情况调整） |

#### 步骤3：设置通知

```
通知方式：
- 邮件：您的邮箱
- 短信：您的手机号（需绑定）
- 旺旺/钉钉：可选
```

---

### 方法2：使用云监控

#### 步骤1：访问云监控

访问：https://cloudmonitor.console.aliyun.com/

#### 步骤2：创建报警规则

1. 点击「报警规则」→「创建报警规则」
2. 关联资源：选择您的OSS Bucket
3. 设置报警条件：

```
指标：公网流出流量
周期：1小时
条件：求和 > 1GB
连续N次：1次
```

## 设置CDN费用预警

### 步骤1：进入CDN监控

1. CDN控制台：https://cdn.console.aliyun.com/
2. 左侧菜单「监控」→「使用量监控」

### 步骤2：设置流量预警

1. 点击「配置报警规则」
2. 配置：

| 配置项 | 推荐值 |
|--------|--------|
| **规则名称** | CDN流量预警 |
| **监控指标** | 流量 |
| **统计周期** | 1小时 |
| **阈值** | 500MB/小时（新手建议设低一点） |
| **报警级别** | 提醒 → 警告 → 紧急 |

### 步骤3：配置通知

```
通知方式：
- 邮件：必填
- 短信：建议开启
- WebHook：可选（可对接钉钉群）
```

## 预警通知配置

### 配置接收方式

#### 1. 邮件通知（必选）

```
接收人：您的常用邮箱
发送频率：立即发送
```

#### 2. 短信通知（建议）

```
条件：紧急预警才发短信
接收手机：您的手机号
费用：免费（有赠送额度）
```

#### 3. 钉钉群通知（可选）

适合团队项目：

```markdown
1. 创建钉钉群
2. 添加自定义机器人
3. 复制WebHook地址
4. 粘贴到阿里云报警通知配置
```

## 额外安全措施

### 1. 设置消费上限（推荐）

阿里云提供「按量付费+SLA」的停机策略：

1. 访问「费用中心」→「费用限制」
2. 设置「可用额度阈值」
3. 设置触发动作：「发送通知」或「停止服务」

### 2. 购买资源包（封顶成本）

```
推荐配置：
- OSS存储包：10GB（10元/年）
- CDN流量包：100GB（25元/年）

总费用：35元/年
即使被攻击，也只消耗资源包额度
```

### 3. 配置Referer防盗链

防止他人盗用您的资源：

1. OSS控制台 → 存储桶 → 「数据安全」→「Referer防盗链」
2. 配置白名单：
   ```
   Referer：https://您的域名.com
   允许空Referer：否（推荐）
   ```

## 常见问题

### Q1：收不到预警通知？

**检查项**：
1. 邮箱是否正确？垃圾箱？
2. 手机号是否已实名认证？
3. 短信额度是否充足？
4. 报警规则是否已启用？

### Q2：如何测试预警是否生效？

**方法**：
1. 临时将阈值调低
2. 产生一次触发条件
3. 验证是否收到通知
4. 记得调回正常阈值

### Q3：费用已达上限怎么办？

**紧急处理**：
1. 立即检查是否有异常流量
2. 联系阿里云客服说明情况
3. 排查攻击来源并封禁IP
4. 考虑开启更高级别的防护
