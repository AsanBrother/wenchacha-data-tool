## CDN基础配置

### 前提条件

- ✅ 已开通CDN服务
- ✅ 已创建OSS Bucket

---

### 步骤1：开通CDN服务

1. 访问阿里云CDN控制台：https://cdn.console.aliyun.com/
2. 点击「立即开通」
3. 选择计费方式：
   - **按量付费**：先用后付，适合测试
   - **资源包**：提前购买，适合正式使用

---

### 步骤2：添加加速域名

1. 在CDN控制台点击「域名管理」→「添加域名」

2. 填写基本信息：

| 配置项 | 填写内容 | 说明 |
|--------|---------|------|
| **加速域名** | 输入您的域名 | 如：`www.yoursite.com`（可选自定义域名） |
| **业务类型** | 选择「静态加速」 | 适合静态网站 |
| **源站类型** | 选择「OSS域名」 | 让CDN回源到OSS |

3. 配置源站信息：

```
源站类型：OSS域名
OSS域名：选择您的Bucket
地域：选择与您OSS相同的地域
```

4. 点击「下一步」等待审核（通常几分钟）

---

### 步骤3：配置CNAME

添加域名后，CDN会提供CNAME地址：

```
示例：
CDN分配的CNAME：xxx.w.kunlungr.com
```

**在DNS服务商添加CNAME记录**：

| 记录类型 | 主机记录 | 记录值 |
|---------|---------|--------|
| CNAME | www | xxx.w.kunlungr.com |

⚠️ **等待时间**：CNAME配置后需要10-30分钟生效

## CDN + OSS联动配置

### 为什么需要联动？

```
用户请求流程（无CDN）：
用户 → 直接请求OSS → 每次都从OSS获取

用户请求流程（有CDN）：
用户 → CDN缓存 → 命中则直接返回
     ↓ 未命中
     CDN → 回源OSS获取 → 缓存 → 返回给用户
```

---

### 配置步骤

#### 步骤1：回源配置

1. 进入CDN域名管理
2. 点击域名右侧「配置」→ 「回源配置」
3. 设置：

```
回源协议：跟随访问协议（推荐）
回源HOST：开启并设置为您的OSS域名
私有Bucket回源：否（公共读Bucket不需要）
```

#### 步骤2：缓存配置

1. 点击「缓存配置」→ 「缓存过期时间」
2. 添加缓存规则：

| 缓存内容 | 缓存时间 | 说明 |
|---------|---------|------|
| HTML文件 | 1小时 | 内容可能频繁更新 |
| CSS/JS | 1天 | 7天 |
| 图片 | 1个月 | 30天 |
| 其他 | 1周 | 7天 |

#### 步骤3：设置Referer防盗链

1. 点击「访问控制」→ 「Referer防盗链」
2. 配置：

```
类型：白名单
Referer：填写您的域名
允许空Referer：否（推荐）
```

## HTTPS配置

### 为什么需要HTTPS？

- 🔒 数据传输加密
- 🔍 身份验证
- 📱 微信小程序/APP要求HTTPS
- 🔍 SEO优化（百度收录）

---

### 步骤1：上传SSL证书

#### 方法A：使用阿里云SSL证书（推荐）

1. 访问SSL证书控制台：https://yundun.console.aliyun.com/
2. 购买或申请免费证书
3. 点击「部署」→ 选择CDN

#### 方法B：上传自有证书

1. 准备证书文件：
   - 证书文件（.pem或.crt）
   - 私钥文件（.key）

2. 在CDN控制台点击「域名管理」→ 「HTTPS配置」→ 「上传证书」
3. 填写证书信息并上传

---

### 步骤2：开启HTTPS

1. 在CDN域名管理中点击域名「配置」
2. 找到「HTTPS配置」
3. 点击「修改」

```
HTTPS证书：选择已上传的证书
HTTP/2：开启（推荐）
强制跳转：HTTPS（推荐）
TLS版本：TLS 1.2 + TLS 1.3
```

---

### 步骤3：配置HSTS（可选）

增加安全性：

```
开启HSTS：否（可选）
HSTS时间：31536000（1年）
包含子域名：否
预加载：否
```

⚠️ **注意**：HSTS开启后无法撤销，请谨慎配置

## 性能优化配置

### 页面优化

#### 1. 开启页面优化

1. 「性能优化」→ 「页面优化」
2. 开启：
   - ✅ HTML优化（压缩HTML）
   - ✅ CSS优化（压缩CSS）
   - ✅ JS优化（压缩JS）

#### 2. 开启Gzip压缩

1. 「性能优化」→ 「智能压缩」
2. 选择压缩类型：
   ```
   压缩类型：Brotli压缩（优先） + Gzip压缩（备用）
   文件类型：text/html, text/css, application/javascript等
   ```

---

### 图片优化

#### 1. 开启图片压缩

1. 「性能优化」→ 「图片优化」
2. 开启：
   - WebP自适应
   - 图片压缩（可选）

#### 2. 配置图片处理

```
格式转换：WebP优先
质量压缩：开启（质量85%）
尺寸限制：最大4096px
```

---

### 访问优化

#### 1. 设置并发限制

1. 「访问控制」→ 「单请求并发数」
2. 设置合理的并发限制防止滥用：
   ```
   单IP最大连接数：100
   单URL最大连接数：50
   ```

#### 2. 配置带宽上限（防攻击）

1. 「用量管理」→ 「带宽封顶」
2. 设置：

```
带宽上限：100Mbps（根据实际需求）
触发动作：监控 + 关闭加速服务
```

## 安全防护配置

### DDoS基础防护

阿里云CDN自带基础DDoS防护：
- 免费基础防护：2Gbps
- 超过会自动清洗

---

### CC防护配置

防止恶意CC攻击：

1. 「安全防护」→ 「CC防护」
2. 配置：

```
CC防护：开启
防护模式：观察
自定义规则：可添加
  - 规则名称：限制高频访问
  - 匹配条件：URI + 频率
  - 限制动作：验证码或拦截
```

---

### IP黑名单

阻止恶意IP访问：

1. 「访问控制」→ 「IP黑名单」
2. 添加恶意IP：
   ```
   IP地址：1.2.3.4
   生效时间：永久
   ```

---

### Referer防盗链

防止资源被盗用：

1. 「访问控制」→ 「Referer防盗链」
2. 配置：

```
类型：白名单
Referer列表：
  - https://yourdomain.com
  - https://www.yourdomain.com
允许空Referer：否
```

## 常见问题

### Q1：CDN加速不生效？

**排查步骤**：
1. 检查CNAME是否配置正确
2. 使用 `nslookup` 验证CNAME：
   ```bash
   nslookup -type=cname 您的CDN域名
   ```
3. 等待10-30分钟生效
4. 检查本地DNS缓存：`ipconfig /flushdns`

---

### Q2：部分文件不缓存？

**原因**：可能是以下情况
- 文件设置了 `Cache-Control: no-cache`
- 文件在CDN不支持的类型列表中
- 源站返回了 `Set-Cookie` 头

**解决方法**：
1. 检查源站HTTP头设置
2. 在CDN缓存配置中强制缓存特定文件

---

### Q3：如何查看CDN命中率？

1. CDN控制台 → 「数据分析」→ 「CDN命中率」
2. 查看：
   - 请求命中率
   - 流量命中率

**优化目标**：
- 请求命中率：> 90%
- 流量命中率：> 95%

---

### Q4：如何清除CDN缓存？

#### 方法A：URL刷新

1. 「缓存刷新」→ 「URL刷新」
2. 输入要刷新的URL：
   ```
   https://www.yoursite.com/index.html
   ```

#### 方法B：目录刷新

1. 「缓存刷新」→ 「目录刷新」
2. 输入目录：
   ```
   https://www.yoursite.com/static/
   ```

⚠️ **限制**：每天有刷新额度限制

---

### Q5：HTTPS证书过期怎么办？

1. 提前30天续费证书
2. 在CDN控制台更新证书
3. 确保证书配置正确后部署
