## 准备工作

### 需要的工具和账号

- ✅ 阿里云账号（已实名认证）
- ✅ 对象存储OSS服务已开通
- ✅ AccessKey ID 和 AccessKey Secret

### 获取AccessKey

1. 登录阿里云控制台
2. 访问：https://ram.console.aliyun.com/manage/ak
3. 点击「创建AccessKey」
4. 完成手机验证
5. **立即保存** AccessKey ID 和 AccessKey Secret

⚠️ **重要提示**：
- AccessKey Secret只显示一次，请立即保存！
- 建议创建RAM子账号并授予最小权限

## 创建OSS存储桶

### 步骤1：进入OSS控制台

访问：https://oss.console.aliyun.com/

### 步骤2：创建存储桶

1. 点击「创建存储桶」
2. 填写配置信息：

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| **存储桶名称** | 填写唯一名称，如 `my-data-tool` | 全球唯一，不能重复 |
| **地域** | 选择离您近的区域 | 如华东1（杭州） |
| **存储类型** | 标准存储 | 网站文件需要频繁访问 |
| **冗余类型** | 本地冗余 | 价格便宜，足够稳定 |
| **读写权限** | 公共读 | 网站需要公开访问 |

### 步骤3：确认创建

点击「确定」完成创建。

## 配置静态网站托管

### 为什么需要配置？

静态网站托管让您可以通过OSS直接访问网站，无需配合云服务器。

### 配置步骤

1. 进入创建的存储桶
2. 左侧菜单找到「基础设置」→「静态网站托管」
3. 点击「设置」
4. 配置如下：

```
默认首页：index.html
默认 404 页：index.html
```

⚠️ **重要**：单页应用（如React/Vue项目）需要将404页也设置为index.html，以便正确处理路由。

## 部署网站文件

### 方法1：使用部署脚本（推荐）

#### 1. 授权阿里云CLI

```bash
# 在终端运行
aliyun configure

# 按提示输入：
#   AccessKey ID: 您的AccessKey ID
#   AccessKey Secret: 您的AccessKey Secret
#   Default Region: 您的区域（如cn-hangzhou）
#   Default Output Format: json
```

#### 2. 修改部署脚本

编辑 `deploy-oss.sh`，修改以下配置：

```bash
OSS_BUCKET="您的存储桶名称"
OSS_REGION="cn-hangzhou"  # 您的区域
```

#### 3. 运行部署脚本

```bash
# 添加执行权限
chmod +x deploy-oss.sh

# 运行部署
./deploy-oss.sh
```

脚本会自动：
- 安装依赖
- 构建项目
- 上传文件到OSS
- 配置静态网站托管

---

### 方法2：手动上传

#### 1. 构建项目

```bash
npm install
npm run build
```

#### 2. 上传文件

在OSS控制台：
1. 进入存储桶
2. 点击「文件管理」
3. 点击「上传文件」
4. 选择 `dist` 文件夹中的所有文件
5. 确认上传

#### 3. 设置文件访问权限

上传后，选中所有文件 → 点击「更多」→「设置HTTP头」
```
Content-Type: application/octet-stream（对于.js文件）
Content-Type: text/html（对于.html文件）
Cache-Control: max-age=31536000（可选，用于缓存）
```

## 配置CDN加速

CDN可以显著提升网站访问速度并减轻OSS压力。

### CDN优势

- ✅ 访问速度更快（就近访问）
- ✅ 减轻源站压力
- ✅ 抵御小规模DDoS攻击
- ✅ 节省OSS流量费用

### 配置步骤

#### 步骤1：开通CDN服务

1. 访问阿里云CDN控制台：https://cdn.console.aliyun.com/
2. 点击「立即开通」
3. 选择「按量付费」或购买资源包

#### 步骤2：添加加速域名

1. 在CDN控制台点击「添加域名」
2. 配置信息：

| 配置项 | 填写内容 |
|--------|---------|
| **加速域名** | 输入您的域名（可选，如不需要自定义域名可跳过） |
| **业务类型** | 选择「静态加速」 |
| **源站类型** | 选择「OSS域名」 |
| **OSS内网域名** | 选择您的OSS Bucket |

#### 步骤3：配置CNAME

1. 添加域名后，CDN会提供CNAME地址
2. 复制CNAME地址
3. 在您的域名DNS设置中添加CNAME记录：

```
记录类型：CNAME
主机记录：www（或@）
记录值：粘贴CDN提供的CNAME地址
解析线路：默认
TTL：10分钟
```

⚠️ **注意**：添加CNAME可能需要10-30分钟生效

#### 步骤4：验证配置

访问CDN提供的加速地址，验证网站是否正常访问。

## 常见问题

### Q1：网站访问显示403 Forbidden

**原因**：存储桶权限不是「公共读」

**解决方法**：
1. 进入存储桶设置
2. 「基础设置」→「权限管理」
3. 修改为「公共读」

---

### Q2：网站访问显示404

**原因**：静态网站托管未配置或配置错误

**解决方法**：
1. 进入存储桶「静态网站托管」设置
2. 确认默认首页和404页都设置为 `index.html`

---

### Q3：文件已更新但访问的还是旧内容

**原因**：浏览器缓存

**解决方法**：
1. 强制刷新：Ctrl + Shift + R（Windows）或 Command + Shift + R（Mac）
2. 或清除浏览器缓存
3. 或在URL后加版本号：`index.html?v=2`

---

### Q4：CDN加速不生效

**原因**：CNAME未生效或配置错误

**解决方法**：
1. 检查CNAME是否配置正确
2. 使用DNS查询工具验证：`nslookup -type=cname 您的CDN域名`
3. 等待CNAME生效（最多30分钟）

---

### Q5：担心被恶意刷流量产生高额费用

**解决方法**：
1. 设置费用预警（见费用预警设置指南）
2. 购买流量包封顶成本
3. 开启CDN的带宽封顶功能
4. 配置Referer访问控制
